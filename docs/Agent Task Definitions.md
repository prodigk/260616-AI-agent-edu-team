# Agent Task Definitions

본 파일은 대시보드 및 리포트 생성 워크플로우에서 각 에이전트가 수행해야 할 작업을 정의합니다. Python 스크립트나 LangChain과 연동하여 멀티 에이전트 시스템을 구현할 때 참고할 수 있습니다.

## 1. Data Loader Agent

**목적:** 여러 CSV 파일을 불러와 사용 가능한 통합 데이터 구조를 만듭니다.

**주요 작업:**

1. `sales_data.csv`, `product_data.csv`, `marketing_data.csv`, `customer_data.csv`, `inventory_data.csv`, `review_cs_data.csv`, `recommendation_log.csv` 파일을 읽습니다.
2. `date` 컬럼을 날짜형으로 변환합니다.
3. `product_id`를 기준으로 판매 데이터와 재고, 리뷰, 추천 로그를 병합할 수 있도록 데이터프레임 구조를 정리합니다.
4. 데이터의 누락값, 이상치 등을 확인하고 필요하면 보정합니다.
5. 통합된 데이터 구조(예: 딕셔너리 또는 클래스)를 다음 에이전트에게 전달합니다.

## 2. Event Detector Agent

**목적:** 통합 데이터에서 의미 있는 이벤트를 탐지합니다.

**주요 작업:**

1. 최근 N일의 이동 평균과 표준편차를 계산하여 이상 징후를 판단합니다. 예: 3일 평균 대비 50% 이상 판매 증가 → `판매 급상승` 이벤트.
2. 재고가 `safe_stock` 미만일 경우 `품절 위험`, 재고가 0이면 `품절` 이벤트를 생성합니다.
3. 광고 데이터에서 ROAS가 전주 대비 20% 이상 감소하면 `광고 효율 저하` 이벤트를 생성합니다.
4. 각 카테고리별 판매량 상위 3위 안에 들어온 상품을 `인기상품 진입` 이벤트로 표시합니다.
5. 추천 로그에서 클릭률 또는 전환율이 평균 대비 30% 이상 높으면 `추천상품 성과 우수` 이벤트를 생성합니다.
6. 생성된 이벤트 객체에는 이벤트명, 대상 상품, 발생 날짜, 관련 지표(판매량 변화율 등)를 포함합니다.

## 3. Correlation Analyzer Agent

**목적:** 이벤트와 지표 사이의 상관관계를 분석하여 근본 원인 및 영향 요소를 파악합니다.

**주요 작업:**

1. 통합 데이터에서 주요 지표(`quantity_sold`, `ad_spend`, `clicks`, `conversions`, `revenue`, `current_stock`, `rating`, `recommended_count`, `click_count`, `conversion_count`)의 피어슨/스피어만 상관계수를 계산합니다. 기간별 필터(예: 최근 7일, 14일)를 적용해 시계열의 영향을 고려합니다.
2. 상관관계 행렬을 생성하고 상관계수가 절대값 기준으로 높은 상위 N쌍을 추출합니다.
3. 이벤트 발생일 주변(예: ±1일)의 지표 변화를 분석하여 특정 이벤트와 특정 지표 사이의 상관성을 평가합니다. 예를 들어 `판매 급상승` 이벤트 전후 광고비와 클릭수의 변화.
4. 분석 결과를 네트워크 데이터 구조(노드: 이벤트/지표, 엣지: 상관계수) 형태로 출력해 시각화 에이전트에서 사용하도록 합니다.

## 4. Insight Generator Agent

**목적:** 이벤트와 상관관계 분석 결과를 바탕으로 자연어로 인사이트를 생성합니다.

**주요 작업:**

1. Event Detector Agent와 Correlation Analyzer Agent가 생성한 데이터 구조를 입력으로 받아 이번 주 혹은 선택 기간의 핵심 이슈와 원인을 3~5개 문장으로 요약합니다.
2. 상관관계가 높은 지표 쌍을 토대로 추정되는 원인과 영향을 설명합니다. 예: "감성 무드등 판매 급상승은 인스타그램 광고 클릭수 급증과 강한 양(+)의 상관관계를 보였습니다." 등.
3. 인사이트는 각 역할(마케팅, MD, CRM/CS, 대표)이 이해하기 쉬운 언어로 작성하며, 다음 단계에서 액션플랜 생성에 활용됩니다.

## 5. Weekly Report Agent

**목적:** 지정한 주간 범위에 대한 리포트를 생성합니다.

**주요 작업:**

1. 한 주간의 KPI(매출, 주문, 광고 성과, 신규·재구매 고객, 리뷰 평균 등)를 요약합니다.
2. Event Detector Agent가 탐지한 이벤트 목록과 Correlation Analyzer Agent의 상관관계 Top N을 포함합니다.
3. Insight Generator Agent의 요약을 리포트 섹션으로 포함합니다.
4. 표와 그래프(예: KPI 추세 그래프, 상관관계 히트맵, 이벤트 타임라인)를 생성하여 첨부합니다.
5. 마케팅팀, MD팀, CRM/CS팀, 대표 등 역할별 섹션을 만들어 각 역할이 집중해야 할 사항을 정리합니다.
6. 리포트는 Markdown 또는 PDF로 저장되어 전달되며, 명확한 제목과 날짜를 포함해야 합니다.

## 6. Action Planner Agent

**목적:** 상관관계 분석 및 이벤트 정보를 기반으로 실행 가능한 액션을 생성합니다.

**주요 작업:**

1. 각 이벤트에 대해 예상되는 원인과 지표의 상관성을 참조하여 해결책을 도출합니다. 예: `판매 급상승`과 `재고 부족`의 상관성이 높다면 추가 발주 권고.
2. 각 액션에 대해 **담당팀**, **우선순위(H/M/L)**, **마감일**, **세부 지침**을 정의합니다.
3. 액션 목록을 JSON 또는 DataFrame 형태로 정리하여 Task Assignment Agent에 전달합니다.
4. 우선순위는 상관계수의 절대값, 이벤트 영향도, 긴급성 등을 기반으로 계산합니다.

## 7. Task Assignment Agent

**목적:** 생성된 액션을 팀과 담당자에게 배정하고 상태를 관리합니다.

**주요 작업:**

1. Action Planner Agent로부터 받은 액션 목록을 역할별(마케팅, MD, CRM/CS, 운영, 대표)로 필터링합니다.
2. 각 역할별로 액션을 담당자에게 배정하고, 진행 상태(`대기`, `진행중`, `완료`)를 초기화합니다.
3. 액션 보드 형태의 DataFrame이나 데이터베이스 테이블을 유지하여 대시보드에서 조회할 수 있도록 합니다.
4. 마감일이 지난 액션이나 미완료 항목을 리포트에 표시하는 알림 기능을 제공합니다.

## 8. Executive Report Agent

**목적:** 대표 또는 최고 결정권자가 확인할 최종 요약 보고서를 작성합니다.

**주요 작업:**

1. Weekly Report Agent의 결과를 기반으로 하되, 핵심 KPI, 주요 이벤트, 상관분석 결과, 가장 중요한 액션 항목을 간결하게 정리합니다.
2. 대표가 승인해야 할 항목(예: 추가 발주, 광고 예산 조정)을 명확히 표시하고, 의사결정에 필요한 데이터를 첨부합니다.
3. 보고서는 보통 1~2페이지 분량으로 작성하며, 이해하기 쉬운 서식과 시각 자료를 사용합니다.

---

이 파일에 정의된 작업을 기반으로 에이전트 설계를 진행하면, 데이터 로딩에서 이벤트 탐지, 상관관계 분석, 인사이트 생성, 리포트 작성, 액션플랜 도출, 역할 배정, 그리고 최종 보고서까지 일관된 워크플로우를 구현할 수 있습니다.
