# 개발 계획 업데이트 및 데이터 재생성 안내

## 1. LLM 기반 커머스 대시보드 최신 기능 요약

최근 리서치에 따르면 AI와 LLM을 접목한 대시보드는 단순 분석을 넘어 **실행 가능한 의사결정 지원**으로 발전하고 있습니다. 주요 트렌드는 다음과 같습니다.

1. **자연어 인터페이스 (NLQ/NLG)** – 사용자가 자연어로 질문하면 모델이 SQL을 생성하여 차트와 보고서를 그려 주며, 지표 변화에 대한 내러티브 요약을 자동 작성합니다. 이는 비개발자의 데이터 접근성을 높여 줍니다.
2. **실시간 데이터 및 AI 알림** – 라이브 데이터와 연결해 KPI를 실시간 반영하고, 매출 급락·광고비 급등 등 이상 징후를 탐지하여 이메일/Slack 알림을 전송합니다.
3. **맞춤형 KPI와 역할 기반 뷰** – 조직 목표에 따라 KPI를 커스터마이징하고, CFO·CMO 등 역할별로 다른 지표를 표시합니다. 세밀한 권한 제어와 행 단위 접근 관리가 강조됩니다.
4. **인터랙티브·예측 시각화** – 클릭과 드릴다운을 통해 세부 원인을 탐색하고, 트렌드 예측 및 What‑If 시나리오를 통해 미래 변화를 미리 시뮬레이션합니다.
5. **AI 기반 상관관계·원인 분석** – 판매 급상승, ROAS 하락 등의 이벤트를 다른 지표와 연결해 원인 후보를 제시하고, 고객 세그먼트·이탈 예측·감성 분석을 수행합니다.
6. **내러티브 요약과 권고** – AI가 보고서를 요약하면서 왜 변화가 발생했는지 설명하고, 이후 취해야 할 액션까지 제안합니다. 이는 관리자 업무를 크게 줄여 줍니다.
7. **거버넌스와 트레이싱** – 정확한 메트릭 정의를 포함한 세만틱 레이어를 사용하고, 모든 프롬프트와 SQL 쌍을 기록해 정확도를 평가합니다.
8. **AI 대시보드 커머스 사례** – Search Atlas의 LLM Visibility 도구는 브랜드가 LLM에서 어떻게 언급되는지 분석하고, Medusa Admin은 프롬프트 기반으로 즉석에서 대시보드를 생성하는 등 다양한 구현 사례가 나타나고 있습니다.

## 2. 기존 MVP와의 시너지 및 적용 방안

현재 MVP는 CSV 데이터를 기반으로 이벤트 탐지 → 상관관계 분석 → 인사이트 도출 → 액션플랜 → 보고서 작성으로 이어지는 구조입니다. 최신 사례를 참고해 다음과 같은 기능을 우선 도입할 수 있습니다:

- **자연어 질의와 차트 생성** – 채팅 패널에 NLQ 기능을 통합하여 “이번 달 ROAS 추세를 보여줘”와 같은 질문을 하면 적절한 필터와 시각화를 자동 생성합니다. 실패 시 규칙 기반 답변으로 대체합니다.
- **AI 기반 이상 탐지와 알림** – 규칙 기반 이벤트 탐지에 통계적 이상치 탐지를 추가하고, 심각한 경우 Slack/이메일 알림을 보냅니다. AI는 이상 원인을 간단히 설명합니다.
- **예측 분석과 What‑If 시나리오** – 단순 예측 모델을 활용하여 매출·재고·광고 효율의 미래 추세와 가정 변화 시 영향을 추정합니다. 사용자 입력 슬라이더를 통해 가정을 바꿀 수 있게 합니다.
- **내러티브 리포트와 역할별 권고 강화** – 보고서에서 KPI 변동 이유를 설명하고, 역할별(마케팅·재고·CS 등) 구체적 행동 지침을 제공합니다.
- **인터랙티브 상관 네트워크** – 상관관계 히트맵을 보완하여 노드‑링크 형태의 네트워크 그래프를 도입하고, 클릭 시 상세 추세와 원인을 보여줍니다. 상관관계와 인과관계를 구분하는 안내 문구를 포함합니다.
- **LLM 거버넌스 기능 도입** – KPI 정의를 별도 파일로 관리하고, 프롬프트 및 SQL 로그를 기록·모니터링하여 응답 품질을 평가합니다.

위 기능 중 **자연어 질의/차트 생성**, **AI 이상 탐지**, **내러티브 리포트 강화**는 기존 구조와 잘 맞아 빠르게 도입할 수 있으므로 우선순위로 제안합니다.

## 3. CSV 데이터 재생성 계획

새로운 기능 개발과 Q&A 테스트를 위해 **데모용 CSV 데이터를 새로 생성**할 필요가 있습니다. 다음과 같은 가정을 두고 데이터를 재생성할 예정입니다.

1. **기간과 범위** – 최근 90일 간의 데이터를 생성합니다. 날짜는 `date` 컬럼으로 하루 단위입니다.
2. **도메인별 스키마 유지** – 기존 `sales_data.csv`, `product_data.csv`, `marketing_data.csv`, `customer_data.csv`, `inventory_data.csv`, `review_cs_data.csv`, `recommendation_log.csv`의 컬럼 구조를 그대로 유지합니다.
3. **값 생성 로직**
   - 매출·주문·광고 지표는 임의의 정규분포 및 요일/시즌 효과를 반영하여 생성합니다.
   - 제품 ID는 `product_data.csv`에 정의된 항목을 사용하며, 각 제품의 가격·안전재고 등의 정보를 고정합니다.
   - 고객·CS 지표는 신규 고객 수, 재구매율, 평점, 문의 건수 등을 현실적인 범위에서 무작위로 생성합니다.
   - 추천 로그는 추천 노출수, 클릭수, 전환수를 평균 CTR/CVR을 기반으로 계산하여 생성합니다.
4. **재사용과 자동화** – Python 스크립트(`generate_fake_data.py`)를 작성해 반복적으로 데이터를 생성할 수 있도록 하며, 향후 질문 응답과 기능 테스트에 활용합니다.

위 계획을 바탕으로, 개발팀은 CSV 재생성 스크립트를 작성하고 새로운 기능 구현 전후로 데이터 셋을 갱신하여 테스트할 수 있습니다.
